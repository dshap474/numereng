# Write Contract

The goal is decision usefulness.

## CURRENT.md

Must include:

- summary snapshot
- current promoted defaults
- active frontier
- what we are not doing
- top 5 ranked next experiments
- recent material shifts
- key evidence anchors

Each next-experiment card must include:

- title
- hypothesis
- why now
- expected information gain
- expected upside
- success criteria
- minimal run plan
- evidence links

## Experiment reviews

Must include:

- summary
- experiment question and surface
- key results
- what this experiment establishes
- what remains ambiguous
- comparison to relevant prior evidence
- global contradiction / support check
- impact on frontier
- addenda

Reviews are mostly immutable.
Later material changes append a dated addendum.

## Topic ledgers

Use a hybrid structure:

- current state
- promoted beliefs / defaults
- anti-patterns / invalidated beliefs
- open questions
- evidence log

## Decision notes

Create only when the frontier materially changes.

Must include:

- decision
- prior state
- new state
- why it changed
- evidence
- consequences

## Links

- use note-relative links for notes under `__RESEARCH_MEMORY__`
- use explicit app routes for experiment navigation
- avoid raw filesystem paths unless no app route exists
